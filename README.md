## 📖 Overview (概要)
**「看護師の離職は、個人の問題だけなのか？」**

私は10年以上看護師として臨床現場に立ち、多くの同僚が辞めていくのを見てきました。現場では離職理由が属人的に語られがちですが、私は「地域特性や労働市場などの**構造的な要因**も大きく影響しているのではないか？」という仮説を持ちました。

本プロジェクトは、47都道府県の公開データを用いた**定量分析（PCA・クラスタリング）**により、従来の定説（残業や給与が主因である等）を検証し、**地域構造（都市 vs 地方）ごとの真の離職要因**を特定したものです。
また、単なる分析に留まらず、再現性を担保するための**ETLパイプライン構築**など、データエンジニアリングの観点も取り入れています。

## 🛠 Engineering Focus (データエンジニアリング的側面)
本プロジェクトでは、DEとしての実務を想定し、以下の観点を重視して実装しました。

* **Data Quality**: 外れ値検知と統計的な妥当性確認によるデータの信頼性担保。
* **Modular Code**: 分析処理を関数化し、再利用性とメンテナンス性の高いコード構成を追求。
* **Environment Independence**: 相対パス管理と依存ライブラリの明示による、環境を選ばない再現性の確保。

## 🔍 Key Findings (分析結果サマリ)

### 1. 定説の崩壊と真因の特定
重回帰分析の結果、一般的に信じられている「残業時間」や「賞与」は全国一律の離職要因ではないことが判明しました。
代わりに、以下の2つが統計的に有意な「2大要因」として特定されました。

> * **家賃相場 (Rent)**: 都市部における生活コストの圧迫（係数: 正）
> * **夜勤負担 (Night Shift)**: 地方部における過重労働（係数: 正）

### 2. 日本の医療圏の「4分類」構造
デンドログラム分析により、全国は明確に異なる課題を持つ4つのグループに分類されました。

| グループ名 | 地域特性 | 課題と特徴 |
| :--- | :--- | :--- |
| **Group 1（都市部）** | 東京・大阪など | 家賃が高く、生活苦による離職が多い。 |
| **Group 2（ホワイト地方）** | 岩手・富山など | **離職率 7.3%**（夜勤負担が極めて少ない理想モデル）。 |
| **Group 3（地方課題）** | 地方全般 | 人手不足で夜勤が集中し疲弊（夜勤72h超率: 38.6%）。 |
| **Group 4（準・都市）** | 中核都市 | コスト高と激務の二重苦。 |

### 3. 地方における「賃上げ」の限界
地方エリアに限定した分析では、**「年収」と「離職率」には相関がほぼ見られない（相関係数: 0.096）** 一方で、「夜勤負担」との相関は強い（0.271）ことが判明しました。
これにより、地方においては「金銭」よりも「労働環境（休息）」が重要であることが示唆されました。

## 🚑 Policy Simulation (政策提言)

モデルを用い、「都市部への家賃補助」と「地方部への夜勤改善」を行った場合の離職率改善効果を試算しました。

| ターゲット地域 | 最適な処方箋 (Action) | 改善効果 (ROI) |
| :--- | :--- | :--- |
| **A. 大都市圏** | **💰 家賃補助の拡充**<br>(月額2万円の支給を想定) | **離職率 ▼ 2.00 pt**<br>(11.97% → 9.97%) |
| **B. 地方・中核都市** | **🌙 夜勤体制の改革**<br>(72h超率を20%以下へ抑制) | **離職率 ▼ 1.76 pt**<br>(9.34% → 7.58%) |

**結論**: 全国一律の施策ではなく、**都市部は「経済的支援」、地方部は「人員配置の見直し」** という、地域特性に合わせたリソース配分が最も効果的です。

## 🛠 Technology & Engineering (技術スタック)
本プロジェクトでは、分析の再現性と保守性を担保するため、以下の技術選定とディレクトリ構成を採用しています。

* **Languages**: Python 3.10+, SQL
* **Libraries**: Pandas, Scikit-learn, Matplotlib/Seaborn
* **Data Engineering (ETL)**:
    * **Pre-processing**: Excel/PDF形式の元データをPython (`00_data_preprocessing.ipynb`) でクレンジング・結合し、分析用マートを作成。
    * **Validation**: データの欠損値処理および型定義の厳密化。
